# 🛠 یادداشت فیکس‌ها (نسخه دوم) — PDF و سیستم Quota

## ۱) PDF: متن نامفهوم → خواندن مستقیم با Gemini

### ریشه مشکل
کد قبلی متن PDF رو با یه regex ساده استخراج می‌کرد:
```php
preg_match_all('/\((.*?)\)\s*Tj/s', $content, ...)
```
این روش برای PDF‌های قدیمی ساده کار می‌کنه، **ولی برای PDF فارسی** (که از فونت embed با CMap و FlateDecode استفاده می‌کنه) فقط بایت‌های نامفهومی استخراج می‌کنه. وقتی این بایت‌ها به AI فرستاده می‌شد، AI درست می‌گفت «این متن نامفهوم است».

### راه‌حل
به‌جای استخراج متن سمت سرور، **خود PDF رو** به‌صورت base64 به AI می‌فرستیم. Gemini 2.5 Flash از طریق GapGPT (OpenAI-compatible) به‌خوبی PDF رو می‌خونه:

```json
{
  "type": "image_url",
  "image_url": {
    "url": "data:application/pdf;base64,..."
  }
}
```

### محدودیت‌های جدید
- حداکثر حجم PDF: 20MB (برای جلوگیری از timeout)
- Gemini تا 1000 صفحه در یک PDF رو می‌خونه
- اگه PDF خیلی بزرگ بود، پیام مناسب نمایش داده می‌شه

### فایل‌های تغییر یافته
- `api/chat.php` — منطق PDF: متن استخراج → base64 + image_url
- `includes/ai.php` — اضافه شدن `application/pdf` به whitelist + پرامپت سیستمی برای PDF

## ۲) سیستم Quota: بازنویسی کامل

### مشکلات قبلی
1. **`current_user()` با static cache** — بعد از `reserve_message_quota` مقدار stale می‌موند.
2. **Fallback اشتباه به free** — وقتی اشتراک کاربر منقضی می‌شد، کد به سهمیه رایگان برمی‌گشت. این یعنی کاربر اشتراکی که پلن خریده، بعد از انقضا یواشکی روی پلن رایگان می‌رفت بدون اطلاع.
3. **پیام خطای کلی** — کاربر فقط می‌دید «سقف پیام شما پر شده»، نمی‌فهمید چرا و چی کار باید بکنه.
4. **race condition** — دو تب باز با هم می‌تونستن یه پیام اضافی بفرستن.
5. **Quota در حالت abort برنمی‌گشت** — اگه کاربر وسط استریم تب رو می‌بست، سهمیه‌اش کم می‌شد ولی پیامی save نمی‌شد.
6. **نمایش sidebar اشتباه** — کاربر منقضی می‌دید «پلن رایگان».

### راه‌حل‌ها

#### الف) `current_user($fresh=false)` + تابع جدید `current_user_fresh()`
```php
$user = current_user_fresh(); // cache رو دور می‌زنه و از DB می‌خونه
```

#### ب) `subscription_status()` بازنویسی شد
حالا فیلدهای دقیقی برمی‌گردونه:
```php
[
  'active'    => bool,
  'reason'    => 'متن خطا',
  'plan'      => [...],       // همیشه (اگه پلن داشته)
  'has_plan'  => bool,
  'expired'   => bool,        // اگه منقضی شده
  'limit_hit' => 'daily'|'total'|null,
  'starts_in' => int seconds  // اگه هنوز شروع نشده
]
```

#### ج) `can_send_message()` با ۴ حالت دقیق
```php
// 1) اشتراک فعال
['ok'=>true, 'mode'=>'subscription', 'plan'=>..., 'remaining'=>5]

// 2) اشتراک منقضی/پر (فالبک به free نمی‌ره!)
['ok'=>false, 'mode'=>'subscription_blocked', 'limit_hit'=>'daily', 'plan'=>...]

// 3) سهمیه رایگان فعال
['ok'=>true, 'mode'=>'free', 'remaining'=>2]

// 4) سهمیه رایگان پر
['ok'=>false, 'mode'=>'free_blocked']
```

#### د) `reserve_message_quota()` به‌صورت atomic در یک UPDATE
```sql
UPDATE users SET messages_used_today = messages_used_today + 1, ...
WHERE id = ?
  AND subscription_type = ?         -- مطمئن می‌شیم پلن عوض نشده
  AND subscription_end > NOW()      -- منقضی نشده
  AND messages_used_today < daily_limit
  AND messages_used_total < total_limit
```
اگه `rowCount === 1` → موفق. اگه 0 → یعنی شرطی نقض شده (مثلاً تب دیگه‌ای پیشدستی کرد) و کاربر پیام block رو می‌بینه.

#### هـ) Refund خودکار در سه حالت
1. AI خطا داد
2. Garbage تولید کرد
3. کاربر وسط استریم تب رو بست (`connection_aborted()`)

#### و) پیام Block به‌صورت SSE داخل bubble چت
وقتی quota پره، به‌جای 403 خشک، یه پیام فارسی زیبا داخل bubble assistant استریم می‌شه:

```
**⏱ اشتراک ۳ ساعته به پایان رسیده است**

برای ادامه استفاده از دانش‌یار، اشتراکت رو تمدید کن.

👈 [تمدید اشتراک](https://yoursite.com/pricing.php)
```

پیام‌های اختصاصی برای هر حالت:
- اشتراک منقضی → تمدید
- سقف روزانه پر → «فردا ساعت ۰۰:۰۰ ریست می‌شه» + پیشنهاد ارتقاء
- سقف کل پر → «خرید پلن جدید»
- سهمیه رایگان پر → تشویق خرید پلن
- اشتراک هنوز شروع نشده → نمایش زمان باقی‌مانده

#### ز) Sidebar `chat.php` بازنویسی شد
حالا حالت‌های جدید رو با رنگ‌های مختلف نشون می‌ده:
- 🟢 فعال (سبز)
- 🔴 منقضی/پر (قرمز)
- 🔵 هنوز شروع نشده (آبی)
- ⚪ پلن رایگان (خنثی)

## تست کن

### تست PDF
1. یه PDF فارسی (مثل یه فصل کتاب) بفرست
2. ✅ باید AI متن واقعی رو ببینه و خلاصه/پاسخ بده
3. اگه نگفت «نامفهوم»، یعنی فیکس کار کرده

### تست Quota
**اشتراک منقضی:**
1. در phpMyAdmin برو به جدول `users`
2. یه کاربر تستی رو ست کن: `subscription_end = '2020-01-01 00:00:00'`
3. لاگین کن و یه پیام بفرست
4. ✅ باید پیام «اشتراک به پایان رسیده» با لینک تمدید ببینی

**سقف روزانه پر:**
1. کاربر با پلن weekly: `messages_used_today = 50, daily_limit = 50`
2. پیام بفرست
3. ✅ باید پیام «سقف امروز پر شد، فردا ریست می‌شه» ببینی

**Race condition:**
1. دو تب باز کن
2. در هر دو یه پیام بنویس
3. تقریباً همزمان enter بزن
4. ✅ یکی می‌فرسته، دیگری پیام block می‌بینه (نه دو پیام موفق)

**Abort:**
1. یه پیام طولانی بفرست
2. وسط استریم F5 بزن یا تب رو ببند
3. کاربرت رو در DB چک کن: `messages_used_today` نباید زیاد شده باشه

## نکته نصب

بعد از آپلود این فایل‌ها روی هاست:
1. **Ctrl+F5** کن (cache version از v=19 به v=20 تغییر کرده)
2. تست‌های بالا رو انجام بده
3. اگه قبلاً ادمین در `users` پر شده، نیازی به migration نیست — فیلدها تغییر نکردن

اگه از قبل کاربر با `subscription_end` آینده داشتی، بعد از این آپدیت کارش دقیقاً مثل قبل اجرا می‌شه — فقط در صورت تموم شدن، رفتار درست‌تری می‌بینه.
