# 🎯 فیکس‌های نسخه ۴ — کیفیت فرمول و پاسخ AI

## ۳ مشکلی که فیکس شد

### ۱) فرمول‌های inline در متن دیده نمی‌شدن
**قبل:** `.katex` فقط `font-size: 1.08em` بود و در میان متن فارسی گم می‌شد.

**حالا:** هر فرمول inline یه pill نارنجی ملایم می‌گیره با border و رنگ متمایز (`#ffd6a8`):
```css
.message .bubble .katex {
  background: linear-gradient(180deg, rgba(235,124,42,.10), rgba(235,124,42,.05));
  border: 1px solid rgba(235,124,42,.18);
  border-radius: 6px;
  padding: 1px 7px;
  color: #ffd6a8;
}
```

### ۲) AI گاهی فرمول رو خراب می‌نوشت (مثل `ρA>ρB`)
**ریشه:** پرامپت قبلی فقط می‌گفت «از LaTeX استفاده کن» ولی صریح نمی‌گفت همیشه. مدل گاهی برای ساده‌نویسی از unicode + markdown italic استفاده می‌کرد که منجر به آشغال‌هایی مثل `ρA*ρA*​>*ρB*​` می‌شد.

**حالا ۳ لایه دفاع گذاشتم:**

#### لایه ۱: پرامپت سیستمی بازنویسی شده (در `includes/ai.php`)
قوانین جدید سختگیر:
- 🔴 **قانون طلایی:** هر عبارت ریاضی/فیزیکی/شیمیایی - حتی یک متغیر تک‌حرفی - باید **حتماً** داخل LaTeX باشد
- نمونه‌های دقیق درست و غلط با علامت ✅ و ❌
- مثال‌های ملموس از نحوه نوشتن فرمول‌های پایه دبیرستان

#### لایه ۲: تنظیمات مدل بهینه شد
- `temperature`: ۰.۳ → **۰.۲** (دقیق‌تر، کمتر خلاقیت بی‌جا)
- `max_tokens`: ۴۰۰۰ → **۶۰۰۰** (فضای کافی برای پاسخ کامل گام‌به‌گام)
- `top_p`: ۰.۹ → **۰.۸۵** (پایدارتر)

#### لایه ۳: `cleanAiOutput()` در JS
اگه مدل با وجود همه این‌ها باز خراب نوشت، JS قبل از render سعی می‌کنه ترمیمش کنه:
- حذف کاراکترهای ZWSP/ZWJ/BOM زائد
- تشخیص الگوی `*X*​*X*` (که از تکرار اشتباه می‌آد) و رفعش
- تشخیص متغیرهای یونانی خام (مثل `ρA > ρB`) و **خودکار** تبدیل به `$\rho_A > \rho_B$`

### ۳) کیفیت پاسخ AI ضعیف بود
**ریشه:** پرامپت قبلی فقط می‌گفت «صمیمی باش» و «گام‌به‌گام». مدل اغلب جواب کوتاه می‌داد.

**حالا پرامپت ساختار آموزشی واضح می‌خواد:**
```
1. صورت سوال را با کلمات خودت دوباره بنویس
2. داده‌های مسئله را فهرست کن (معلوم‌ها و مجهول‌ها)
3. مفهوم/فرمول کلیدی را با یک جمله توضیح بده
4. حل را گام‌به‌گام بنویس (هر گام شماره‌گذاری شود)
5. در پایان جواب نهایی را با ✅ یا داخل کادر نمایش بده
```

علاوه بر این:
- "هرگز جواب کوتاه و سرسری نده. اگر سوال ساده بود هم حداقل ۳-۴ خط توضیح بده."
- "اگر سوال مبهم بود، قبل از حل، یک سوال شفاف‌کننده بپرس."
- استفاده از ایموجی‌های ۆآموزشی در جای مناسب (📚 ✅ ⚠️ 💡 🎯)

## 💎 پیشنهاد ارتقاء مدل برای کیفیت بهتر

الان مدل `gemini-2.5-flash` استفاده می‌شه که سریع و ارزون ولی متوسطه. اگه می‌خوای کیفیت پاسخ‌ها خیلی بهتر بشه:

در فایل `includes/config.php` خط:
```php
define('AI_MODEL', 'gemini-2.5-flash');
```
رو به یکی از این‌ها تغییر بده:

| مدل | کیفیت | سرعت | هزینه |
|------|--------|--------|--------|
| `gemini-2.5-flash` | ⭐⭐⭐ | ⚡⚡⚡ | 💰 |
| `gemini-2.5-pro` | ⭐⭐⭐⭐⭐ | ⚡⚡ | 💰💰💰 |
| `claude-sonnet-4.5` | ⭐⭐⭐⭐⭐ | ⚡⚡ | 💰💰💰 |
| `gpt-5` یا `gpt-5-mini` | ⭐⭐⭐⭐⭐ | ⚡⚡ | 💰💰💰 |

برای پاسخ‌های آموزشی فارسی، **`gemini-2.5-pro`** بهترین گزینه‌ست (هم قوی هم نسبتاً ارزون).

## 🧪 تست

### پیش‌نمایش بصری
فایل `preview-formula.html` رو در workspace باز کن — ۶ نمونه فرمول مختلف می‌بینی:
1. فرمول inline در متن
2. فرمول بلوک (display)
3. معادله درجه دوم
4. واکنش شیمیایی
5. ماتریس
6. یک حل کامل گام‌به‌گام

### تست در سایت
بعد از آپلود فایل‌ها و Ctrl+F5:
1. **سوال ساده:** "چگالی چیست؟" → باید با $\rho = \frac{m}{V}$ شروع کنه و توضیح بده
2. **سوال مقایسه‌ای:** "اگر چگالی A بیشتر از B باشد، یعنی چه؟" → باید با `$\rho_A > \rho_B$` نشون بده، نه `ρA>ρB`
3. **سوال محاسباتی:** "اگر $F = 20\,\text{N}$ و $m = 5\,\text{kg}$، شتاب چقدر است؟" → باید با ۵ گام جواب کامل بده
4. **روی موبایل** هم باز کن و چک کن فرمول‌های inline در متن گم نشن
